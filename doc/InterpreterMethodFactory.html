atom(regex, butNotopt, interpretationopt) → {external:InterpreterObject#atomTypeInterpreterMethod}

The atom interpreter method factory takes a regular expression and makes an external:InterpreterObject#atomTypeInterpreterMethod meant to be an interpreter method of a user created external:InterpreterObject. The returned method parses text with the supplied regular expression. Optionally, an {external:ThisBinding#atomInterpretation} callback function may be supplied to describe how the parsed text should be interpreted. The interpretation will be run as if it was a method of the interpreter object, i. e. this will be bound to the interpreter object inside the interpretation body.
Parameters:
Name Type Attributes Description
regex RegExp The regular expression that the interpreter method should use to parse text.
butNot Array.<string> <optional>
 A list of forbidden words. If the parsed string equals an element in this list, the parsing fails.
interpretation external:ThisBinding#atomInterpretation <optional>
 A callback function describing how the parsed string should be interpreted. If pressent, the result of this function will also be the result of the interpreter method. Otherwise the parsed string will be returned.

empty(interpretation) → {external:InterpreterObject#emptyTypeInterpreterMethod}

The empty method factory takes an interpretation callback function and returns an external:InterpreterObject#emptyTypeInterpreterMethod meant to be a method of a user created external:InterpreterObject. The returned method is a type of external:InterpreterObject#interpreterMethod that doesn't parse anything but just runs the interpretation as i it was a method of the interpreter object, i. e. with this bound to the interpreter object inside the body of the interpretation. The result of the interpretation function will also be the result of the interpreter method. Empty type interpreter methods are mainly useful as a setup function when placed as the first part of an InterpreterMethodFactory#group.

group(…parsable, interpretationopt) → {external:InterpreterObject#groupTypeInterpreterMethod}

The group interpreter method factory accepts any number of interpreterMethodNames and regular expressions in any order as arguments and creates a external:InterpreterObject#groupTypeInterpreterMethod meant to be a method of an external:InterpreterObject. The returned method parses the supplied regular expressions and the external:InterpreterObject#interpreterMethods named by the interpreter method names, in the specified order. It is possible to describe how the returned method should interpret the results of its parts by supplying an external:ThisBinding#groupInterpretation callback function. The interpretation will be run as a method of the interpreter object, i. e. this will be bound to the interpreter object in the body of the interpretation. If an interpretation is supplied, the result of the interpretation will also be the result of the metod. Otherwise the method will return an object with properties with the same name as the names of the parts that holds the results of the respective parts.

 A name of an interpreter metod on the same object or a regular expression that should be parsed.

 A callback function that will be called with the results of the parts of the method. If pressent, the result of the interpretation will also be the result of the interpreter method. Otherwise it will return an object with the result of its parts.

methodFactory(partName) → {external:InterpreterObject#methodFactoryTypeInterpreterMethod}

This method takes an interpreterMethodName and returns an external:InterpreterObject#methodFactoryTypeInterpreterMethod meant to be a method of an external:InterpreterObject. When called, a method factory type interpreter method returns a function that does exactely the same thing as the named part would have done, but with the possibility to run the descendant interpretations with another external:ThisBinding. This type of external:InterpreterObject#interpreterMethods are useful for implementing e. g. functions or loops, for setting another object then the interpreter as the this binding and for running the parsed instruction many times without having to parse it each time.
